# Interaction Specification: 第1回「データはどこから来るのか」

## 1. ページの中心体験

このページでは、学生が「データ表」を見る前の世界に戻る。そこにはまだ数値はなく、`X_1, X_2, ..., X_N` という確率変数だけがある。再生ボタンを押すと、確率分布から砂粒が落ち、1つずつ実現値 `x_i` に変わる。

2変量では、同じことを `(X_i, Y_i)` について行う。砂粒は同時分布から落ち、`(x_i, y_i)` というペアを1行のデータとして作る。

## 2. 単変量 widget（部品）

### 2.1 状態

| 状態 | 意味 |
|---|---|
| `distribution` | 正規分布、一様分布、歪んだ分布、二峰分布 |
| `N` | サンプルサイズ |
| `seed` | 乱数 seed（種） |
| `samples` | 実現される予定の値の配列 |
| `realizedCount` | すでに実現した個数 |
| `status` | `idle`, `playing`, `paused`, `done` |
| `speed` | 再生速度 |

### 2.2 初期表示

- 分布の曲線を表示する。
- 表には `X_i` が並ぶ。
- `実現後` の列には `未実現` と表示する。
- 砂粒は表示しない。

### 2.3 再生

1. `再生` を押す。
2. 次の実現予定値 `samples[realizedCount]` を読む。
3. その値に対応する横位置へ砂粒を落とす。
4. アニメーション完了後、表の該当行を `x_i = 値` に変える。
5. 横軸上に実現済み点を残す。
6. `realizedCount` を1増やす。
7. `realizedCount === N` になったら `done` にする。

### 2.4 停止

- `停止` を押すと、現在の `realizedCount` を保持する。
- すでに実現した点と表の値は残す。
- 再度 `再生` を押すと次の点から続ける。

### 2.5 リセット

- `リセット` を押すと `realizedCount` を0に戻す。
- seed（種）が同じなら、同じ順番で再生される。
- 分布、`N`、seed（種）を変更した場合もリセットする。

### 2.6 一括実現

- `一括実現` を押すと `realizedCount = N` にする。
- アニメーションなしで表をすべて埋める。

### 2.7 表示オプション

- ヒストグラムは実現済みの値だけで作る。
- 標本平均は実現済みの値だけで計算する。
- 初期状態ではヒストグラムや平均は控えめに表示し、主役を「実現」にする。

## 3. 2変量 widget（部品）

### 3.1 状態

| 状態 | 意味 |
|---|---|
| `mode` | `surface` または `contour` |
| `rho` | 相関係数 `ρ` |
| `N` | サンプルサイズ |
| `seed` | 乱数 seed（種） |
| `samples` | 実現される予定の点 `(x_i, y_i)` |
| `realizedCount` | すでに実現したペアの個数 |
| `status` | `idle`, `playing`, `paused`, `done` |
| `speed` | 再生速度 |

### 3.2 表示モード

#### 3D surface（3次元曲面）モード

- 同時分布を山の形で表示する。
- 横方向を `X`、奥行きを `Y`、高さを密度とする。
- 実現済み点は曲面上または曲面近くに表示する。
- 現在落ちている砂粒は、上から対象点へ落ちるように見せる。

#### contour（等高線）モード

- 同じ同時分布を上から見た等高線で表示する。
- 実現済み点は散布図として表示する。
- 現在落ちている砂粒は、対象点へ向かって落ちるように表示する。

### 3.3 モード切り替え

- `surface` と `contour` を切り替えても、`samples` と `realizedCount` は保持する。
- つまり、表示を切り替えても表の値と実現済み点は変わらない。
- これにより、学生は「山」と「等高線」が同じ同時分布を表すことを理解できる。

### 3.4 相関操作

- `ρ` の範囲は `-0.9` から `0.9` とする。
- `ρ = 0` では等高線が円に近く、点の雲に傾きは出にくい。
- `ρ > 0` では右上がりの雲になる。
- `ρ < 0` では右下がりの雲になる。
- `ρ` を変更した場合は、サンプル列を再生成し、`realizedCount` を0に戻す。

### 3.5 表

初期状態の表は次の形式にする。

| i | 実現前 | 実現後 |
|---|---|---|
| 1 | `(X_1, Y_1)` | 未実現 |
| 2 | `(X_2, Y_2)` | 未実現 |

実現後は次のようにする。

| i | 実現前 | 実現後 |
|---|---|---|
| 1 | `(X_1, Y_1)` | `(x_1, y_1) = (0.31, -0.84)` |

## 4. 教材メッセージ

各widget（部品）には、短い観察メッセージを添える。

### 単変量

> 右の表は最初から数字だったわけではありません。各行の `X_i` が実現して、`x_i` になった結果です。

### 2変量

> 2つの列は別々に生まれるのではありません。1つの行として `(X_i, Y_i)` が同時に実現し、`(x_i, y_i)` になります。

## 5. エッジケース

- `N` が大きい場合、表はスクロール領域にする。
- `N` が大きい場合、すべての点を描くが、重くなるなら点の半径を小さくする。
- 3D表示が使えない環境では、等高線モードを初期表示にしてもよい。
- `prefers-reduced-motion`（動きを減らす設定）が有効な場合、砂粒の移動を短縮または省略する。
