# Interaction Spec: Lesson 4 Estimator Quality

## 1. Page identity

```text
Title: 第4回：よい平均の当て方はどれか
Route: /04-estimator-quality/
Main component: EstimatorQualityWidget
```

このページは、第2回「点から分布を読むゲーム」と第3回「母集団分布と標本」の後に置く。現在の順序は次である。

```text
01: データはどこから来るのか
02: 点から分布を読むゲーム
03: 母集団分布と標本
04: よい平均の当て方はどれか
```

Codexは第2回と第3回を入れ替えない。

## 2. Purpose

このページの目的は、期待値 `E[X]` を推定する複数の方法を比較し、推定量の「良さ」を1回の当たり外れではなく、次の2つの性質として観察させることである。

1. 不偏性: 同じ `N` で標本を何度も取り直したとき、推定値の中心が `E[X]` に重なるか。
2. 一致性: `N` を大きくしたとき、推定値が `E[X]` に近づいていくか。

学生向けの期待値表記は必ず `E[X]` とする。ターゲット記号としてギリシャ文字のミューは使わない。

## 3. Learning goals

学生はこのページを通じて、次を理解する。

1. 同じ標本から作れる推定量はいくつもある。
2. 1回のデータで真の値に近かったかどうかだけでは、推定量の良さは判断できない。
3. 不偏性は、同じ `N` で標本を取り直したときの中心を見る性質である。
4. 一致性は、`N` を大きくしたときの挙動を見る性質である。
5. 不偏性があっても一致性がない推定量がある。
6. すべてのデータを使っても、不偏性も一致性も持たない推定量がある。
7. 標本平均は、期待値 `E[X]` の推定量として、不偏性と一致性の両方を持つ代表例である。

## 4. Target quantity

ターゲットは母集団の期待値である。

```text
Target: E[X]
```

UIでは次のように表現する。

```text
本当の期待値 E[X]
```

次の表現は使わない。

```text
母平均
真の平均を表すギリシャ文字のミュー
```

必要に応じて「期待値」と「平均」は併記してよい。

```text
期待値 E[X]、つまり長い目で見た平均
```

## 5. Estimators

比較する推定量は4つで固定する。

| id | 学生向けラベル | 式 | 使うデータ | 不偏性 | 一致性 | 教材上の役割 |
|---|---|---|---|---|---|---|
| `first` | 最初だけ見る | `X_1` | 最初の1個だけ | あり | なし | 不偏性と一致性が別物であることを示す |
| `ends` | 端だけ見る | `(X_1 + X_N) / 2` | 最初と最後だけ | あり | なし | `N` が大きくても実際に使う情報が少ないと安定しないことを示す |
| `mean` | 全員を同じ重みで見る | `(X_1 + ... + X_N) / N` | すべて | あり | あり | 標本平均の良さを示す |
| `rootSumSquares` | 二乗和ルート推定量 | `√(X_1^2 + ... + X_N^2) / N` | すべて | なし | あり | 不偏ではないが、現在の `E[X]=0` の設定ではNを大きくすると0へ近づくことを示す |

4つ目の推定量は、有限Nでは正の値になりやすいが、Nを大きくすると `E[X]=0` に近づく例として扱う。

## 6. Main interaction flow

1. 母集団分布を選ぶ。
2. `N` と seed（種）を設定する。
3. 1つの標本を表示する。
4. その標本から4つの推定量を計算し、横軸上に表示する。
5. `N` を大きくしたときの推定量の軌跡を見る。
6. 同じ `N` で標本を何度も取り直し、推定値の分布を見る。
7. 性質カードを開き、不偏性と一致性の言葉に接続する。

## 7. Display modes

### 7.1 growingN mode

表示名:

```text
Nを大きくする
```

問い:

```text
データを増やすと、どの推定量が E[X] に近づくか？
```

図:

```text
横軸: N
縦軸: 推定値
基準線: E[X]
線: 4つの推定量
```

期待される見え方:

- `first` は `X_1` から動かない。
- `ends` は最後の観測値 `X_N` に反応して揺れ続ける。
- `mean` は `E[X]` の近くに寄る。
- `rootSumSquares` は有限Nでは正の側に出やすいが、Nを大きくすると `E[X]=0` に近づく。

### 7.2 resampling mode

表示名:

```text
標本を取り直す
```

問い:

```text
同じ N で何度も標本を取り直すと、推定値の中心は E[X] に重なるか？
```

図:

```text
横軸: 推定値
縦軸: 推定量
点: 各リサンプリングで得られた推定値
基準線: E[X]
印: 推定量ごとのリサンプリング平均
```

注意:

このリサンプリングは bootstrap（ブートストラップ）ではない。同じ母集団分布から標本を取り直すシミュレーションである。UIでは「標本を取り直す」と書く。

### 7.3 compare mode

表示名:

```text
2つを並べる
```

問い:

```text
一致性と不偏性は、どの図を見ると区別できるか？
```

図:

- 左: `N` を大きくしたときの軌跡。
- 右: 同じ `N` で標本を取り直したときの分布。

## 8. Controls

| UI | 初期値 | 候補 | 説明 |
|---|---:|---|---|
| 母集団分布 | 正規分布 | 正規、右に歪んだ分布、二山型 | 推定量の性質を別の分布でも見る |
| `N` | 30 | 2, 5, 10, 30, 100, 300, 1000 | 固定サンプルサイズ |
| 最大 `N` | 1000 | 30, 100, 300, 1000 | 軌跡表示の上限 |
| リサンプリング回数 `B` | 500 | 100, 500, 1000 | 標本を取り直す回数 |
| seed（種） | 123 | 数値入力 | 再現性 |
| 表示モード | `compare` | `growingN`, `resampling`, `compare` | 図の切り替え |
| 推定量表示 | 全部ON | ON/OFF | 見たい推定量だけ表示 |
| 性質カード | OFF | ON/OFF | 不偏性・一致性を表示 |

## 9. Population distributions

初期実装では3種類でよい。

| id | 表示名 | 期待値 `E[X]` | 目的 |
|---|---|---|---|
| `normal` | 正規分布 | 0 | 基本形 |
| `rightSkewed` | 右に歪んだ分布 | 0に標準化して表示 | 歪んだ分布でも性質を見る |
| `bimodal` | 二山型分布 | 0に標準化して表示 | 混合分布でも期待値を推定することを見る |

すべての分布について、表示上の期待値 `E[X]` を明示する。第4回の主役は推定量なので、分布の種類を増やしすぎない。

## 10. Data and state model

推奨する型:

```ts
type PopulationType = "normal" | "rightSkewed" | "bimodal";

type EstimatorId = "first" | "ends" | "mean" | "rootSumSquares";

type ViewMode = "growingN" | "resampling" | "compare";

type EstimatorQualityState = {
  populationType: PopulationType;
  targetExpectation: number;
  spread: number;
  n: number;
  maxN: number;
  resampleCount: number;
  seed: number;
  selectedEstimators: EstimatorId[];
  viewMode: ViewMode;
  showPropertyCards: boolean;
};
```

`targetExpectation` は `E[X]` の数値である。`mu` という識別子は使わない。

推定量定義:

```ts
type EstimatorDefinition = {
  id: EstimatorId;
  labelJa: string;
  formulaJa: string;
  usesAllData: boolean;
  unbiased: boolean;
  consistent: boolean;
  explanationJa: string;
  compute: (sample: number[]) => number;
};
```

## 11. Visual design

### 11.1 Current sample plot

表示:

- 横軸上に標本点を rug plot（ラグプロット）として表示する。
- 点の重なりを避けるため、縦方向に小さな jitter（微小な揺らし）を入れてよい。
- `E[X]` を縦線で表示する。
- 4つの推定値を異なるマーカーで表示する。

### 11.2 Trajectory plot

表示:

- 横軸は `N`。
- 縦軸は推定値。
- `E[X]` を水平線で表示する。

### 11.3 Resampling distribution plot

表示:

- 横軸は推定値。
- 縦方向に推定量を並べる。
- 各リサンプリング結果を点で表示する。
- 推定量ごとのリサンプリング平均を大きめの印で表示する。
- `E[X]` を縦線で表示する。

## 12. Property cards

性質カードは最初から見せず、学生が図を観察したあとに開けるようにする。

表示例:

```text
理由: 何度も標本を取り直すと平均的には E[X] に重なるが、N を増やしても X_1 だけを見続けるため。
```

```text
二乗和ルート推定量
不偏性: なし
```


## 14. Confirmation questions

1. `X_1` だけを見る推定量は、なぜ不偏性はあるのに一致性がないのでしょうか。
2. `(X_1 + X_N) / 2` は、なぜ `N` が大きくなっても安定しないのでしょうか。
3. 標本平均は、他の推定量と比べて何がよいのでしょうか。
4. `√(X_1^2 + ... + X_N^2) / N` は、なぜ不偏ではないのに、このページではNを大きくすると `E[X]` に近づくのでしょうか。
5. 不偏性と一致性は、それぞれどの図を見ると区別しやすいでしょうか。

## 15. Must / Should / Could

### Must

- 4つの推定量を計算できる。
- `E[X]` の基準線を表示できる。
- `N` を大きくしたときの推定量の軌跡を表示できる。
- 同じ `N` で標本を取り直したときの推定値分布を表示できる。
- seed（種）で再現できる。
- `rootSumSquares` が不偏性なし・一致性ありの例として見えることを確認する。

### Should

- リサンプリング平均、bias（バイアス）、variance（分散）、MSE（平均二乗誤差）を表示できる。
- 推定量のON/OFFを切り替えられる。
- `compare` モードを持つ。

### Could

- 軌跡をアニメーション再生できる。
- 授業中に「どれがよいと思うか」を投票できる。
- 外れ値がある母集団分布を追加する。
