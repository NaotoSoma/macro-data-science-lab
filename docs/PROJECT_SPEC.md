# Project Specification

## 1. 概要

本プロジェクトは、経済学部向けデータサイエンス授業の companion website（補助ウェブサイト）である。各ページは、授業で扱う概念を直接説明する長文ページではなく、1つまたは少数の interactive widget（対話型部品）を中心にした小さな概念実験室として設計する。

序盤では、統計学の根本的な世界観を双方向から扱う。

```text
第1回: 確率分布 → 確率変数の実現 → データ点
第3回: データ点 → 背後にありそうな確率分布の推測
```

第1回の中心命題は次である。

> データは、確率変数が実現した値の集まりである。

第3回ではその逆向きに、目の前の有限個のデータ点から、観測されていない母集団分布の形を推測する。これにより、学生は「データは分布から生まれる」という見方と、「データから分布やモデルを考える」という見方の両方を持てるようになる。

## 2. 対象読者

- 経済学部の学部生
- 統計学、計量経済学、データサイエンスを初めて体系的に学ぶ学生
- 数式には抵抗があるが、表、図、操作には反応しやすい学生

## 3. 学習上の到達目標

### 3.1 第1回の到達目標

学生は第1ページを通じて、次を理解する。

1. 実現前の `X_i` と実現後の `x_i` は違う。
2. サンプルサイズ `N` は、実現される確率変数の個数である。
3. データ表は、確率変数の実現値を行として並べたものである。
4. 2変量データでは、`X_i` と `Y_i` が同時に実現し、1行のペア `(x_i, y_i)` になる。
5. 同時分布は、3D surface（3次元曲面）としても、contour（等高線）としても同じ対象を表している。
6. 相関 `ρ` を変えると、2変量分布の形と観測点の雲の向きが変わる。

### 3.2 第3回「点から分布を読むゲーム」の到達目標

学生は第3ページを通じて、次を理解する。

1. 観測されたデータ点は、背後にある観測されない母集団分布から出てきた有限個の実現値である。
2. `N` が小さいと、真の分布の形ははっきり見えない。
3. `N` を大きくすると、分布の形が見えやすくなる。
4. ヒストグラムの bin（階級）幅を変えると、同じデータでも見え方が変わる。
5. KDE（カーネル密度推定）の bandwidth（帯域幅）を変えると、滑らかさと細部の見え方が変わる。
6. 有限個のデータから母集団分布を一意に当てることはできない。
7. 統計学では、データから「真実そのもの」を読むのではなく、もっともらしい分布やモデルを考える。

## 5. 技術仕様

### 5.1 フレームワーク

Astro（静的サイト生成フレームワーク）を採用する。理由は、各ページの大部分を静的にし、必要な widget（部品）だけをクライアント側で動かせるためである。

### 5.2 ページ記述

MDX（Markdownに部品を埋め込める形式）を採用する。各ページは短い導入文、操作の観点、widget（部品）、確認質問で構成する。

### 5.3 対話型部品

React（UI部品ライブラリ）と TypeScript（型付きJavaScript）で実装する。数学ロジックは純粋関数として切り出し、UI状態とは分離する。

### 5.4 2次元可視化

SVG（2次元ベクター描画）を主に使う。必要に応じて D3.js（データ可視化ライブラリ）を導入するが、初期試作では依存しすぎない。第3回「点から分布を読むゲーム」はSVGで実装する。

### 5.5 3次元可視化

React Three Fiber（React用Three.jsレンダラー）と Three.js（3次元描画ライブラリ）を使う。3D表示は重くなりやすいため、必ず等高線モードを代替表示として維持する。第3回は1変量の推測ゲームなので、3D表示を追加しない。

### 5.6 乱数と再現性

乱数は seed（種）で再現可能にする。第3回では、真の分布の選択、真のパラメータ、観測点をseed（種）から決定論的に生成する。同じseed（種）と同じ `N` なら同じ問題が再現される。

## 6. 第1ページの教材構成

### 6.1 導入

短く次を述べる。

> データは、確率変数が実現した値を表に並べたものです。

ただし、以後の理解は図で進める。導入文は長くしない。

### 6.2 Widget 1: 単変量の確率変数

学生は `N`、分布、seed（種）、再生速度を操作する。図では確率分布から砂粒が落ち、表の `X_i` が `x_i` に変わる。

### 6.3 Widget 2: 2変量の確率変数

学生は `N`、相関 `ρ`、seed（種）、表示モードを操作する。表示モードは 3D surface（3次元曲面）と contour（等高線）を切り替える。

### 6.4 確認質問

ページ末尾には短い確認質問を置く。

1. 実現前の `X_i` と実現後の `x_i` は何が違うか。
2. `N` を増やすと、実現値の集まりは何に近づいて見えるか。
3. 2変量データで `x_i` と `y_i` が同じ行に入るのはなぜか。
4. 3D surface（3次元曲面）と contour（等高線）は何を共有しているか。

## 7. 第3ページ「点から分布を読むゲーム」の教材構成

### 7.1 導入

第3ページは、第1ページの逆向きの体験として設計する。

```text
第1回: 分布が見えている → データ点が出てくる
第3回: データ点だけが見えている → 分布の形を推測する
```

冒頭文は短くする。

> 目の前にある点は、背後にある確率分布から実現した値です。このページでは、点だけを見て、どんな分布から来たのかを推測します。

続けて、次を強調する。

> 同じデータに見えても、背後にある分布は一意には決まりません。だから統計学では、データからもっともらしい分布やモデルを考えます。

### 7.2 Widget: DistributionGuessingGameWidget

新しい widget（部品）は `DistributionGuessingGameWidget.tsx` とする。

主な流れは次である。

1. hidden distribution（隠された分布）をseed（種）から生成する。
2. その分布から `N` 個の観測点を生成する。
3. 初期状態では、点だけを表示する。
4. 学生が候補分布を選び、形を調整する。
5. 学生が「予想を決定」を押す。
6. 学生が「正解を見る」を押す。
7. 真の分布を表示し、予想分布と比較する。
8. ヒストグラム、KDE（カーネル密度推定）、スコア、学びコメントを表示する。
9. 「同じ分布でNを増やす」で、同じ母集団分布からより多くの点を出す。

### 7.3 真の分布

初期実装では、真の分布は次の5種類とする。

| ID | 学生向けの正解名 | 教材上の狙い |
|---|---|---|
| `normal` | 山が1つで左右対称 | 基本形 |
| `uniform` | 平らな分布 | 山がない分布 |
| `rightSkewed` | 右に長い尾 | 所得・資産のような歪み |
| `leftSkewed` | 左に長い尾 | 歪みの向き |
| `bimodal` | 山が2つ | 異なる集団が混ざる状況 |

### 7.4 予想分布

学生が選べる予想分布は、最初は次の4種類とする。

| ID | 学生向けラベル | 操作 |
|---|---|---|
| `singlePeak` | 山が1つ | 中心、広がり |
| `flat` | 平ら | 中心、幅 |
| `skewed` | 片側に長い尾 | 中心、広がり、尾の向き、歪みの強さ |
| `twoPeaks` | 山が2つ | 中心、山の間隔、左右のバランス |

### 7.5 表示レイヤー

| レイヤー | 初期状態 | 目的 |
|---|---|---|
| データ点 | ON | 最初の観察対象 |
| ヒストグラム | OFF | 点の集まりを粗く見る |
| KDE（カーネル密度推定） | OFF | 滑らかな経験的形を見る |
| 予想分布 | 調整中はON | 学生の仮説を表示する |
| 真の分布 | reveal（正解表示）後にON | 答え合わせ |
| 代替候補分布 | 発展モードでON | 複数の分布がありうることを示す |

### 7.6 操作項目

| 操作 | 初期値 | 内容 |
|---|---:|---|
| サンプルサイズ `N` | 30 | 10, 30, 100, 300から選択 |
| 難易度 | 標準 | やさしい、標準、むずかしい |
| seed（種） | 2026 | 同じ問題を再現する |
| 予想タイプ | 山が1つ | 候補分布を選ぶ |
| 中心 | 0 | 曲線を左右に動かす |
| 広がり | 1 | 曲線の幅を変える |
| 歪みの向き | 右 | skewed（歪み型）のみ |
| 歪みの強さ | 1 | skewed（歪み型）のみ |
| 山の間隔 | 2 | twoPeaks（二山型）のみ |
| 左右のバランス | 0.5 | twoPeaks（二山型）のみ |
| ヒストグラム | OFF | 表示切り替え |
| bin（階級）数 | 20 | ヒストグラムの粗さ |
| KDE（カーネル密度推定） | OFF | 表示切り替え |
| bandwidth（帯域幅） | 標準 | KDEの滑らかさ |
| 予想を決定 | - | 予想分布を固定する |
| 正解を見る | - | 真の分布を表示する |
| 同じ分布でNを増やす | - | hidden distribution（隠された分布）は維持する |
| 新しい問題 | - | 真の分布とseed（種）系列を更新する |

### 7.7 確認質問

1. `N` が小さいとき、なぜ真の分布と違う形に見えることがあるか。
2. ヒストグラムの bin（階級）幅を変えると、何が変わるか。
3. KDE（カーネル密度推定）の bandwidth（帯域幅）を変えると、何が変わるか。
4. 二山型の分布が一山型に見えるのは、どのようなときか。
5. データから分布を「当てる」ことと、データから分布を「推測する」ことは何が違うか。

## 8. 非機能要件

- 初期ページ表示は軽くする。
- 3D表示は必要なページでだけ読み込む。
- 低性能PCでは等高線モードに切り替えられるようにする。
- スマートフォンでも操作できるが、授業中の主対象はノートPCまたは大学PCとする。
- UI文言は日本語を基本とする。
- 乱数は seed（種）で再現可能にする。
- 色だけに依存せず、線種、凡例、ラベルで可視化要素を区別する。
- `prefers-reduced-motion`（動きを減らす設定）に配慮する。
