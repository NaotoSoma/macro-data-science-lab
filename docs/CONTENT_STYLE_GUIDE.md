# Content Style Guide

## 1. 基本方針

このwebsite（ウェブサイト）は講義ノートではない。授業の内容を文字で再掲するのではなく、授業中に伝えた直観を interactive figure（対話型図）として体験させる。

## 2. ページ構成

各ページは原則として次の構成にする。

```text
1. 短い問い
2. interactive widget（対話型部品）
3. 観察ポイント
4. 最小限の数式または定義
5. 確認質問
```

## 3. 文体

- 学生に直接語りかける。
- 文を短くする。
- 1ページで伝える概念を1つに絞る。
- 「まず公式を覚える」ではなく「まず何が起きているかを見る」順にする。
- 専門語は使うが、初出時には図の操作と結びつける。

## 4. 避ける表現

避けるべきもの:

- 長い講義再録
- 公式の羅列
- 定義から始まる重い説明
- 図を見なくても完結してしまう本文
- 英語だけのUIラベル

## 5. 推奨する表現

推奨するもの:

- 「まだ値はありません。あるのは `X_i` だけです。」
- 「砂粒が落ちた位置が、今回の `x_i` です。」
- 「この表は、確率変数が1つずつ実現した結果です。」
- 「2つの値は別々ではなく、同じ行として同時に生まれます。」
- 「3Dの山と等高線は、同じ同時分布を別の角度から見ています。」

## 6. 数式の出し方

数式は、学生が図で現象を見たあとに出す。

悪い順序:

```text
X_i \sim F と定義する。標本とは...  
```

良い順序:

```text
表の各行は、最初は `X_i` でした。砂粒が落ちると `x_i` になります。
この関係をまとめて書くと、`X_i \sim F` と表せます。
```

## 7. 用語統一

| 用語 | 使い方 |
|---|---|
| 確率変数 | 実現前の `X_i` |
| 実現値 | 実現後の `x_i` |
| 観測値 | データ表に入った実現値 |
| サンプルサイズ | 実現する行数 `N` |
| 同時分布 | 2変量の分布 |
| 相関 | 2変量分布の傾きや点の雲の向き |
