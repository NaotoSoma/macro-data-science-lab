# Theory Notes: Expectation Estimators

この文書は、第4回「よい平均の当て方はどれか」の実装で、推定量の性質を誤らないためのメモである。学生向けUIでは長い証明を出さない。

## Target

ターゲットは母集団の期待値である。

```text
E[X]
```

コード上の識別子は `targetExpectation` または `expectedValue` とする。`mu` という識別子は使わない。

## Assumption

基本設定では、`X_1, X_2, ..., X_N` は同じ母集団分布から独立に生成されるとする。各 `X_i` の期待値は `E[X]` である。

## Estimator 1: first

```text
first = X_1
```

性質:

- 不偏性: あり。
- 一致性: なし。

理由:

- 何度も標本を取り直すと、`X_1` の平均は `E[X]` に重なる。
- しかし `N` を増やしても `X_1` だけを見続けるため、ブレは小さくならない。

## Estimator 2: ends

```text
ends = (X_1 + X_N) / 2
```

性質:

- 不偏性: あり。
- 一致性: なし。

理由:

- `X_1` と `X_N` はどちらも期待値 `E[X]` を持つので、平均的には `E[X]` に重なる。
- しかし、どれだけ `N` が大きくなっても2個しか使わないため、ブレは消えない。

UIでは `N` の最小値を `2` にする。`N < 2` の場合は定義しない、または `first` と同じ扱いにするが、授業上は `N >= 2` に固定する方がよい。

## Estimator 3: mean

```text
mean = (X_1 + ... + X_N) / N
```

性質:

- 不偏性: あり。
- 一致性: あり。

理由:

- 何度も標本を取り直すと、標本平均の中心は `E[X]` に重なる。
- `N` が大きくなるとブレが小さくなり、`E[X]` の近くに集まる。

## Estimator 4: rootSumSquares

```text
rootSumSquares = sqrt(X_1^2 + ... + X_N^2) / N
```

性質:

- 不偏性: なし。
- 一致性: あり。

理由:

- 二乗してから足すため、有限Nでは正の値になりやすく、推定値の中心は `E[X]` からずれる。
- このページの母集団はすべて `E[X]=0` になるようにしている。
- `sqrt(X_1^2 + ... + X_N^2)` はおおむね `sqrt(N)` のスケールで増えるため、Nで割ると0へ近づく。
- すべてのデータを使っていても、不偏性と一致性は別の性質であることを示せる。

この推定量は、現在の `E[X]=0` という教材設定では一致的に見える。一般の `E[X]` を推定する万能な方法として扱わない。

## Recommended metrics

リサンプリング結果から、次を表示してよい。

```text
resamplingMean = 推定値のリサンプリング平均
bias = resamplingMean - E[X]
variance = 推定値のリサンプリング分散
MSE = bias^2 + variance
```

学生向けには次のように表示する。

| 指標 | 表示名 |
|---|---|
| `bias` | ずれ |
| `variance` | ブレ |
| `MSE` | ずれとブレを合わせた悪さ |

## Visual interpretation

| 概念 | 見る図 | 見方 |
|---|---|---|
| 不偏性 | リサンプリング分布 | 点の集まりの中心が `E[X]` に重なるか |
| 一致性 | `N` を大きくした軌跡 | `N` が増えると `E[X]` に近づくか |
| ブレ | リサンプリング分布の幅 | 標本を取り直すと推定値がどれくらい揺れるか |
| 有限Nでの正のずれ | `rootSumSquares` の中心 | 不偏ではないが、Nを大きくすると `E[X]=0` に近づくか |
